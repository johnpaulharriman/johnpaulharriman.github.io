Background

Prefetching code insertion has limitless possibilities because it is loading memory before it executes at any stage of your code. Computing every possibility would take an exceptional amount of time, and running every possible code would take even longer. We are trying to make this process easier by running on multiple different cores with non-interfering caches.

The aspects of the problem that benefit from this problem are two fold. The first is that we are able to continuously update our machine learning model by continually testing different versions of the prefetching code. The second is that the formation of the code itself can be done in parallel which is then subsequently tested, but generation could be done on a different machine.

Pseudocode example of code distribution:

AllPossibilities = genInParallel(codeFile)
foreach program in AllPossibilities
timeRan = run_prefetch_code(program)
update_model(timeRan, program)

Pseudocode example of code generation:

def gen_prefetch_code(code):
all_programs = []
foreach line in lines(code):
newCode = generate_all_lines(line,code)
new_programs = gen_prefetch_code(newCode)
all_programs += new_programs
return all_programs
