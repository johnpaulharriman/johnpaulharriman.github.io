Goals and Deliverables

There are two aspects to this project: Speedup of program instances in runtime due to the hidden latency costs that prefetching provides, and also the computation of the machine learning model itself. We want this project to be usable for many programs, not just those that are set in stone, therefore we can set a cap to the duration of the computation of the machine learning model as a baseline for the project.

We cannot yet determine specific performance metrics, given that at this point we are merely theorizing the project itself, but we would like the model usable on X lines of code in under Y minutes, with X and Y to be determined in the first two weeks of the project. Our base goal would be to improve the efficiency of the sequential machine learning algorithm, which we can standardize by expecting a speedup of 1.5x over the baseline model. We expect this to be possible because we will be using separate processors to train the model on specific instances and will also be parallelizing the computation of the model on each instance. We have yet to determine the number of processors we will use, but even with the overhead associated with the distribution of instances to cores, we expect that with 2 cores, at least 1.5x speedup will be possible. When it comes to the speedup of the runtime of each program instance, this metric is less relevant given that it is not related to the 15-418 course load. We would still like to see a speedup with the runtime, but for the deliverables of the project we will focus on the speedup achieved on model building. By the presentation of the project we would like to provide speedup graphs over the multiple iterations of the project, beginning from the sequential model to our final model.
